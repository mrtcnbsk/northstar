# EPIC 6 — TUI: Authoring (Builder)

> REQUIRED SUB-SKILL: superpowers:subagent-driven-development. Checkbox steps. TDD.

**Goal:** A `northstar` TUI **Builder** screen (OpenTUI/SolidJS) to author the on-device org from the terminal: a **Models** view (connected providers + models table + "add provider"), an **Agents** editor (library + form: name, mode, model-dropdown, permission toggles, subordinates, toolpacks, role/prompt → writes `.kilo/agent/<name>.md`), and an **Organization** editor (template picker + org graph + pipeline editor + live validation → writes `.kilo/organization.jsonc`).

**Branch:** `feat/tui-builder` (off main `cb8a236c68`).

**Acceptance (exit):** From the Builder, create a zero org (a new agent with `subordinates` + a small pipeline) → the serializers write `.kilo/agent/*.md` + `.kilo/organization.jsonc` → `OrgSchema.loadOrganization` + `validate` + `crossCheck` + the `org_status` dry-run all pass → the written files round-trip byte-faithfully through the loader.

**Architecture (from recon `wf_3c4c7b5e-226`):** New full-screen screen = **KiloClaw route pattern** (`{type:"builder"}` route). Dialogs overlay routes, so the Builder route opens `DialogSelect`/`DialogPrompt` sub-modals for pickers. The **testable cores** are two serializers + the validation wiring; the TUI render is modeled on `dialog-model.tsx` / `dialog-export-options.tsx` / `dialog-console-org.tsx` and is manual-verifiable (no render harness — same posture as EPIC 5.2's wizard and the W3 console).

**Security invariant (PRESERVE):** the Builder writes only NON-secret config to PROJECT `.kilo/` (agent `.md`, `organization.jsonc`). Provider **keys** are added ONLY through EPIC 5.2's "Add a local provider" wizard, which writes to the GLOBAL `auth.json` (0600). The Builder MUST NOT write any provider key/secret to project config. Do NOT touch `config/variable.ts`.

**Conventions:** `bun` at `~/.bun/bin`. Test from `packages/opencode/`: `bun test <path>`. Typecheck `bun turbo typecheck --filter='!@kilocode/kilo-jetbrains'`. NEVER stage `bun.lock`. `// kilocode_change` on shared-file edits (block form for multi-line); new files under `src/kilocode/**` get `// kilocode_change - new file` line 1. Verify guards after each commit: `bun run script/check-opencode-annotations.ts` (real exit on its OWN line — a pipe masks it). Push `--no-verify`.
