Scan files and/or directories for hardcoded secrets committed to source instead of hand-grepping for API keys and tokens.

Given a list of `paths` (files and/or directories), resolves each to the concrete files it should scan: a file path is scanned directly; a directory is recursed into, skipping common non-source directories (`.git`, `node_modules`, `.build`, `DerivedData`, `Pods`) and binary files (detected by extension and by sniffing the first bytes for a null byte). Total bytes scanned across the whole call are capped at 2MB — once the cap is hit, remaining files are skipped and reported via `truncated: true` rather than silently dropped.

For each scanned text file, checks every line for:
- an AWS access key id (`AKIA` followed by 16 uppercase alphanumeric characters)
- a private key header (`-----BEGIN [RSA|EC|OPENSSH|DSA ]PRIVATE KEY-----`)
- a secret-looking variable assignment: a key name matching `(api[_-]?key|secret|token|password|passwd|client[_-]?secret|access[_-]?token)` (case-insensitive) followed by `:` or `=` and a quoted string literal at least 8 characters long — including the quoted-key form `"api_key": "value"` used in JSON and plist dicts
- (config files only) the same key name followed by `:`/`=` and an UNQUOTED bare-token value at least 8 characters long — e.g. `API_KEY=sk-live-...` in a `.env`, `.xcconfig`, `.properties`, `.ini`, `.cfg`, `.conf`, `.sh`/`.bash`/`.zsh`, `.yaml`/`.yml`, or `.toml` file (or a `.env*` dotfile)
- (plist/`.xcprivacy` files) an Apple `<key>secretName</key><string>value</string>` pair whose key name is secret-looking and whose value is at least 8 characters (key and value may be on separate lines)

The unquoted-value check is deliberately SCOPED to config-file extensions: an unquoted secret-ish assignment in ordinary code (e.g. `apiKey = computeKey()` in a `.swift` file) is a variable/function reference, not a committed literal, so flagging it would be a false positive; the trade-off is that a genuinely committed unquoted secret in a non-config file extension is not caught by this pass. The bare-token value charset excludes spaces and parentheses, so a function call is never mistaken for a literal secret.

This is intentionally conservative — it does NOT flag generic high-entropy strings (hashes, UUIDs, base64 blobs) because that produces too many false positives outside a name-assignment or known-prefix context. It also skips obvious placeholder values for every assignment format: empty strings, `xxx...`-style filler, `<...>`-bracketed placeholders, `YOUR_...`-style templates, `changeme`, `todo`, `example`, and interpolation expressions (Swift `\(...)` or shell/JS `${...}`) whose value is computed rather than a literal secret.

Use this before committing, when reviewing a teammate's/agent's diff, or as a pre-flight check on a directory before shipping, to catch secrets that should be in environment variables or a keychain instead of source.

Parameters:
- `paths`: list of file and/or directory paths to scan.

Returns `{ ok, findings, filesScanned, bytesScanned, truncated? }`. `ok` is `true` only when `findings` is empty. Each finding is `{ file, line, kind, snippet }` where `snippet` redacts the secret value itself, showing only the key name and the first/last 2 characters of the value (e.g. `apiKey = "sk***45"`). An unreadable path or file is skipped with a note rather than causing a throw — this tool never crashes.
