Archive an Xcode scheme (`xcodebuild archive`) and return structured results instead of raw xcodebuild output.

Runs `xcodebuild archive` with the provided scheme/workspace/project/configuration flags plus `-archivePath`, then parses the output into `{ ok, status, errorCount, warningCount, errors[], warnings[], archivePath }`. Each error/warning includes file, line, column, severity, and message when xcodebuild reports them in the standard `file:line:col: severity: message` format.

`status` is one of:
- `"invalid_args"`: `archivePath` escapes the project directory, or `extraArgs` contains a disallowed flag/path. xcodebuild is never invoked.
- `"spawn_failed"`: xcodebuild could not be launched at all (missing toolchain, bad cwd).
- `"archive_failed"`: xcodebuild ran but did not produce `** ARCHIVE SUCCEEDED **` (or exited nonzero).
- `"archive_succeeded"`: the archive was produced; `archivePath` echoes the (validated) path it was written to.

Use this instead of the shell tool for archiving: xcodebuild output can run to thousands of lines, and this tool distills it down to the actionable diagnostics plus a path to the full raw log for cases where more detail is needed. This is typically the step before `ipa_export`.

Parameters:
- `scheme`: the Xcode scheme to archive.
- `workspace`: path to an `.xcworkspace`. Mutually exclusive with `project` in typical usage (xcodebuild allows only one).
- `project`: path to an `.xcodeproj`.
- `configuration`: build configuration, defaults to `Release` (xcodebuild's own default for `archive`).
- `archivePath`: where to write the produced `.xcarchive`. Must be a project-relative path, or an absolute path that resolves inside the project directory — a `..` traversal segment or a path resolving outside the project is rejected as `invalid_args`.
- `extraArgs`: additional raw arguments appended to the xcodebuild invocation. Flags that redirect output outside the project (`-derivedDataPath`, `-resultBundlePath`, `-xcconfig`) or any argument that is an absolute path or contains `..` are rejected before xcodebuild is invoked, returning `{ ok: false, status: "invalid_args", error }`.

`rawLogPath` is a best-effort partial log, written ONLY when diagnostics or an archive failure/timeout occur (omitted on a clean, warning-free success). It may be truncated (see `rawLogTruncated`), so treat it as supplementary detail — the structured `errors[]` and `warnings[]` are always complete regardless of how large the archive output was.
